# WebhookSender 软件设计文档（SDD）

**版本**: 1.0  
**日期**: 2026-06-08  
**架构**: MVVM + Clean Architecture (简化版)

---

## 1. 设计目标

- 采用 Jetpack Compose 构建声明式 UI
- 使用 Room 进行本地数据持久化
- 使用 DataStore 管理用户配置
- 使用 OkHttp 进行网络请求
- 遵循 MVVM 架构，UI 逻辑与业务逻辑分离
- 支持 Kotlin Coroutines 异步处理

---

## 2. 系统架构

### 2.1 架构分层

```
┌─────────────────────────────────────────────────────┐
│                    表现层 (UI Layer)                   │
│    MainActivity.kt / MainScreen.kt / SettingsDialog  │
│    SendViewModel.kt / SettingsViewModel.kt           │
├─────────────────────────────────────────────────────┤
│                  业务逻辑层 (Domain)                   │
│           MessageRepository.kt (Repository Pattern)   │
├─────────────────────────────────────────────────────┤
│                    数据层 (Data Layer)                 │
│    Local:    AppDatabase.kt / MessageEntity.kt       │
│              MessageDao.kt / SettingsDataStore.kt      │
│    Remote:   WebhookApiService.kt                    │
└─────────────────────────────────────────────────────┘
```

### 2.2 技术栈

| 层级 | 技术/库 | 用途 |
|------|---------|------|
| UI | Jetpack Compose + Material 3 | 声明式 UI |
| Architecture | ViewModel + StateFlow | 状态管理 |
| 依赖注入 | Manual DI (App Container) | 对象创建和管理 |
| 网络 | OkHttp 4.x | HTTP 请求 |
| 数据库 | Room 2.x | SQLite ORM |
| 配置存储 | DataStore Preferences | 轻量键值存储 |
| 异步 | Kotlin Coroutines | 协程并发 |
| 编译 | KSP (Kotlin Symbol Processing) | Room 注解处理器 |

## 3. 模块设计

### 3.1 UI 模块 (`ui.screen`)

#### MainScreen.kt
- **WebhookAppScreen**: 主入口，包含 Scaffold、导航、设置弹窗
- **SendScreen**: 发送页面，包含输入框、发送按钮、最近历史预览
- **HistoryScreen**: 历史页面，以 LazyColumn 展示消息卡片
- **HistoryCard**: 单条历史消息卡片组件
- **StatusChip**: 发送状态提示组件
- **SettingsDialog**: 底部弹出的 Webhook URL 配置对话框

#### ViewModel

**SendViewModel**
```
Deps: MessageRepository
State Flows:
  - uiState: StateFlow<SendUiState>  // Idle, Loading, Success, Error
  - historyMessages: Flow<List<MessageItem>>
Input: sendMessage(webhookUrl, keyword, content)
```

**SettingsViewModel**
```
Deps: SettingsDataStore
State Flows:
  - webhookUrl: StateFlow<String>
  - saveResult: StateFlow<Boolean>
Input: saveWebhookUrl(url: String)
```

### 3.2 数据层模块 (`data`)

#### 本地数据 (data/local)

**MessageEntity**
- Room @Entity，定义消息表结构

**MessageDao**
- `getAllMessages()`: Flow<List<MessageEntity>> — 监听式查询
- `insertMessage()`: suspend — 插入记录
- `deleteMessage()`: suspend — 删除单条
- `deleteAllMessages()`: suspend — 清空历史

**SettingsDataStore**
- 包装 `DataStore<Preferences>`
- `webhookUrl`: Flow<String>
- `saveWebhookUrl()`: suspend

**AppDatabase**
- Room Database 单例
- 提供 `messageDao()`

#### 远程数据 (data/remote)

**WebhookApiService**
```
Methods:
  - sendMessage(webhookUrl, keyword, content): SendResult
  
Internals:
  - OkHttpClient 实例（连接超时 10s，读取超时 15s）
  - JSON 组装：{keyword, content, text, timestamp}
  - POST 请求
```

**SendResult**
```kotlin
sealed class SendResult {
    data class Success(val responseCode: Int) : SendResult()
    data class Error(val message: String) : SendResult()
}
```

### 3.3 仓库模块 (data/repository)

**MessageRepository**
```
Deps: MessageDao, WebhookApiService

Methods:
  - allMessages: Flow<List<MessageEntity>>
  - sendMessage(webhookUrl, keyword, content): Boolean
    → 调用 ApiService 发送
    → 无论成功/失败，写入数据库
    → 返回成功/失败布尔值
  - deleteMessage(id): suspend
  - clearAllMessages(): suspend
```

## 4. 数据流设计

### 4.1 发送消息数据流

```
用户点击"发送"
    ↓
SendViewModel.sendMessage(url, keyword, content)
    ↓
uiState → Loading
    ↓
MessageRepository.sendMessage()
    ├──→ WebhookApiService.sendMessage()
    │      └── OkHttp POST → 网络请求
    │
    └──→ 构建 MessageEntity (含结果状态)
           └── MessageDao.insertMessage()
                      ↓
uiState ← Success / Error
    ↓
Room 发出数据变更通知
    ↓
SendViewModel.historyMessages Flow 自动更新
    ↓
UI 自动重组 (Compose)
```

### 4.2 历史数据订阅流

```
MessageDao.getAllMessages() --(Flow)-->
    SendViewModel --(StateFlow)-->
        MainScreen (Send/Histroy) --(Compose recomposition)-->
            LazyColumn 自动更新
```

### 4.3 配置数据流

```
SettingsDialog: 用户输入 URL
    ↓
SettingsViewModel.saveWebhookUrl(url)
    ↓
DataStore.edit { preferences[WEBHOOK_URL] = url }
    ↓
DataStore Flow 下发更新
    ↓
SettingsViewModel.webhookUrl StateFlow 更新
    ↓
SendScreen 观察到新 URL → 配置状态指示器变色
```

## 5. 状态管理

### 5.1 UI State 定义

```kotlin
sealed class SendUiState {
    object Idle : SendUiState()
    object Loading : SendUiState()
    data class Success(val message: String) : SendUiState()
    data class Error(val message: String) : SendUiState()
}

data class MessageItem(
    val id: Int,
    val keyword: String,
    val content: String,
    val timeStr: String,  // 格式化后: "yyyy-MM-dd HH:mm:ss"
    val success: Boolean,
    val errorInfo: String?
)
```

### 5.2 状态转换规则

| 动作 | 前状态 | 后状态 | UI 表现 |
|------|--------|--------|---------|
| 点击发送 | Idle | Loading | 按钮变为加载动画，禁用输入 |
| 发送成功 | Loading | Success | 显示绿色成功提示，重置为 Idle |
| 发送失败 | Loading | Error | 显示红色错误提示，重置为 Idle |
| 3 秒自动清零 | Success/Error | Idle | 状态提示消失 |

## 6. 错误处理设计

| 错误类型 | 处理策略 | 用户可见 |
|----------|----------|----------|
| 无网络连接 | IO Exception → Error("网络错误:") | Toast + 红色状态 |
| HTTP 非 200 | 读取 response code + body → Error | Toast + 红色状态 |
| 超时 (10s/15s) | IOException → Error | Toast + 红色状态 |
| URL 未配置 | 前置校验 → Error("请先配置") | Toast + 红色状态 |
| 内容均为空 | 前置校验 → Error("内容不能为空") | Toast + 红色状态 |
| 数据库写入失败 | try-catch → 不影响上层 | 仅 Logcat 日志 |
| 数据库读取失败 | try-catch → 显示空列表 | 空状态提示 |

## 7. 安全设计

- **网络安全**: AndroidManifest 中 `android:usesCleartextTraffic="false"` 强制 HTTPS
- **数据安全**: 所有数据仅存储在本地，不上传第三方
- **权限最小化**: 仅申请 `INTERNET` 和 `ACCESS_NETWORK_STATE`

    content TEXT NOT NULL,

### 9.2 依赖注入策略

采用 **手动容器模式** (Manual DI Container)，不使用 Hilt（减少复杂度）：

```kotlin
class AppContainer(context: Application) {
    val settingsDataStore = SettingsDataStore(context)
    val database = AppDatabase.getDatabase(context)
    val webhookApiService = WebhookApiService()
    val messageRepository = MessageRepository(
        database.messageDao(), webhookApiService
    )
}
```

ViewModel 通过 `ViewModelProvider.Factory` 注入依赖。
