# WebhookSender 使用说明书

**版本**: 1.1  
**适用平台**: Android 8.0+ (API 26+)

---

## 1. 应用简介

**WebhookSender** 是一款轻量级的 Android 工作待办项快捷记录工具。它可以帮助您快速记录工作待办事项，并将关键字、工作内容以及截止时间以 JSON 格式发送到您指定的 Webhook 地址，适用于对接各类通知机器人、消息推送服务和自建 Webhook 服务。

> 💡 **用途**: 本APP用于工作待办项快捷记录。

---

## 2. 安装与启动

### 2.1 安装
1. 从开发环境构建 APK 包（使用 Android Studio 或 Gradle 命令）
2. 将 APK 文件传输到 Android 设备
3. 在设备上打开 APK 进行安装
4. （如系统提示）允许安装未知来源应用

### 2.2 启动
安装完成后，在应用列表中找到 **WebhookSender** 图标并点击启动。

---

## 3. 初次使用

### 3.1 配置 Webhook 地址
首次启动后，请先配置 Webhook 地址：

1. 点击右上角的 **⚙ 设置** 按钮
2. 在弹出的对话框中输入您的 Webhook URL，例如：
   - 企业微信机器人: `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY`
   - 钉钉机器人: `https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN`
   - 飞书机器人: `https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_KEY`
   - 自建服务: `https://your-domain.com/api/webhook`
3. 点击 **保存** 按钮
4. 返回主界面，状态指示器变为蓝色，显示 **"Webhook 已配置"**

> ⚠️ **注意**: 发送消息前必须先配置 Webhook 地址，否则无法发送。

---

## 4. 发送消息

### 4.1 操作流程

1. 在底部导航栏中切换到 **📤 发送** 页面
2. 在 **关键字** 框中输入消息关键字（如一行标题，可选，会自动记住上次发送的关键字）
3. 在 **工作待办项** 框中输入要发送的待办事项详细内容
4. 点击 **截止时间** 框，在弹出的日期选择器中选择截止日期
5. 点击 **📤 发送** 按钮
6. 等待发送完成，查看结果提示

### 4.2 关键字记忆功能
- 每次发送成功后，App 会自动记住您输入的**关键字**
- 下次打开 App 时，关键字输入框会自动填充上一次的关键字
- 您可以随时修改关键字，修改后发送会自动更新为新的关键字

### 4.3 截止时间选择
- 点击"截止时间"输入框，弹出日期选择器
- 选择日期后，输入框显示格式为 `yyyy-MM-dd`
- 截止时间可选，不填则发送内容中不含截止时间
- 若填写了截止时间，发送的 Webhook 内容会自动在正文前拼接 `【截止时间：xxxx-xx-xx】`

### 4.4 发送按钮状态
| 状态 | 含义 | 操作 |
|------|------|------|
| 蓝色可点击 | 可以发送 | 点击发送 |
| 灰色禁点 | Webhook 未配置 | 先点击 ⚙ 配置 |
| 带旋转圈 | 发送中 | 请等待 |

### 4.5 结果提示

| 提示 | 含义 | 建议 |
|------|------|------|
| ✅ 发送成功 | HTTP 200 | 消息已成功送达 |
| ❌ 网络错误 | 无法连接 | 检查网络或 URL |
| ❌ HTTP 403/404 | 服务端拒绝 | 检查 URL 是否正确 |
| ❌ 超时 | 服务端无响应 | 检查服务端是否在线 |

## 5. 查看发送历史

### 5.1 进入历史页面
点击底部导航栏的 **📜 历史** 图标，切换到历史页面。

### 5.2 历史列表说明
每条记录以卡片形式显示：
- **左侧色带**: 绿色 = 成功，红色 = 失败
- **关键字**: 加粗显示（若无则为"（无关键字）"）
- **工作待办项**: 摘要显示，超长的会自动省略
- **截止时间**: 红色小字显示（若无则不显示）
- **时间**: 格式为 `年-月-日 时:分:秒`
- **状态标签**: 绿色"✓ 成功" 或 红色"✗ 错误原因"

### 5.3 历史记录数量
- 历史记录**最多保存10条**
- 当新的记录加入时，最旧的历史记录会自动删除
- 如需保留更多记录，请自行备份

### 5.4 空状态
如果没有发送记录，页面中央会显示：
- 📜 图标 + "暂无发送记录"

---

## 6. 修改 Webhook 地址

1. 点击任意页面上方的 **⚙ 设置** 按钮
2. 在对话框中修改 URL
3. 点击 **保存**

设置弹窗底部会显示当前版本号，例如：**版本：1.0.1**

---

## 7. 数据管理

### 7.1 数据存储位置
| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| Webhook URL | 本地偏好设置 (DataStore) | 随应用备份恢复 |
| 上次关键字 | 本地偏好设置 (DataStore) | 随应用备份恢复 |
| 发送历史 | 本地 SQLite 数据库 (Room) | 不随云备份 |

### 7.2 数据安全
- 所有数据均存储在设备本地
- 不会上传到任何第三方服务器
- Webhook 地址仅保存在您自己的设备上

## 8. 常见问题（FAQ）

**Q: 为什么点击发送按钮没反应？**
> A: 请检查是否已配置 Webhook 地址。若状态指示器为橙色，请点击 ⚙ 设置。

**Q: 提示"发送成功"但对方没收到？**
> A: 请确认 Webhook URL 正确、服务端正常运行，并且服务端已正确解析 JSON 请求体。

**Q: 如何清除历史记录？**
> A: 当前版本历史记录自动管理，最多保留10条。超出后旧记录会自动删除。如需清除所有数据，请前往系统设置 → 应用 → WebhookSender → 清除存储。

**Q: 为什么历史记录变成了10条？**
> A: 为了控制存储占用，App 设计为最多保留最近10条发送记录，超出后自动清理最旧的记录。

**Q: 关键字会自动修改吗？**
> A: 是的。每次发送成功后，App 会将您填写的关键字保存下来。下次打开 App 时自动填充。您可以手动修改，修改后发送会更新保存为新的关键字。

**Q: 支持哪些 Webhook 服务？**
> A: 任何能接收标准 HTTP POST 请求、Content-Type 为 application/json 的服务均可。

**Q: 发送的 JSON 格式是怎样的？**
> A: 请求体格式如下：
```json
{
  "keyword": "您输入的关键字",
  "content": "您输入的工作待办项",
  "deadline": "2026-06-15",
  "text": "【截止时间：2026-06-15】\nkeyword\n工作待办项",
  "timestamp": 1717826400000
}
```
- 若未填写截止时间，deadline 为空字符串，text = "keyword\n工作待办项"
