# WebhookSender 使用说明书

**版本**: 1.1  
**适用平台**: Android 8.0+ (API 26+)

---

## 1. 应用简介

**WebhookSender** 是一款轻量级的 Android 工作待办项快捷记录工具。它可以帮助您快速记录工作待办事项，并将关键字、工作内容以及截止时间以 JSON 格式发送到您指定的 Webhook 地址，适用于对接各类通知机器人、消息推送服务和自建 Webhook 服务。

> 💡 **用途**: 本APP用于工作待办项快捷记录。

---

## 2. 安装与启动

### 2.1 安装
1. 从开发环境构建 APK 包（使用 Android Studio 或 Gradle 命令）
2. 将 APK 文件传输到 Android 设备
3. 在设备上打开 APK 进行安装
4. （如系统提示）允许安装未知来源应用

### 2.2 启动
安装完成后，在应用列表中找到 **WebhookSender** 图标并点击启动。

---

## 3. 初次使用

### 3.1 配置 Webhook 地址
首次启动后，请先配置 Webhook 地址：

1. 点击右上角的 **⚙ 设置** 按钮
2. 在弹出的对话框中输入您的 Webhook URL，例如：
   - 企业微信机器人: `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY`
   - 钉钉机器人: `https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN`
   - 飞书机器人: `https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_KEY`
   - 自建服务: `https://your-domain.com/api/webhook`
3. 点击 **保存** 按钮
4. 返回主界面，状态指示器变为蓝色，显示 **"Webhook 已配置"**

> ⚠️ **注意**: 发送消息前必须先配置 Webhook 地址，否则无法发送。

---

## 4. 发送待办项

### 4.1 操作流程

1. 在底部导航栏中切换到 **📤 发送** 页面
2. 在 **待办项** 框中输入要发送的待办事项详细内容
3. 在 **优先级** 下拉框中选择优先级（P0高优 / P1一般 / P2低优，默认P1一般）
4. 点击 **截止时间** 框（可选），在弹出的日期选择器中选择截止日期
5. 点击 **📤 发送** 按钮
6. 等待发送完成，查看结果提示

> ⚠️ **注意**: 发送待办项前必须先配置 Webhook 地址，并在**发送页面**已正确填写待办内容。

### 4.2 优先级选择
- 点击"优先级"下拉框选择任务优先级
- 优先级带颜色圆点标识：红色 = P0高优，灰色 = P1一般，绿色 = P2低优

### 4.3 截止时间选择
- 点击"截止时间"输入框，弹出日期选择器
- 选择日期后，输入框显示格式为 `yyyy-MM-dd`
- 截止时间可选，不填则发送内容中不含截止时间
- 若填写了截止时间，发送的 Webhook 内容会自动在正文前拼接截止时间信息

### 4.4 发送按钮状态
| 状态 | 含义 | 操作 |
|------|------|------|
| 蓝色可点击 | 可以发送 | 点击发送 |
| 灰色禁点 | 待办项未填写，或Webhook未配置 | 先填写待办项或点击 ⚙ 配置 |
| 带旋转圈 | 发送中 | 请等待 |

### 4.5 结果提示

| 提示 | 含义 | 建议 |
|------|------|------|
| ✅ 发送成功 | HTTP 200 | 消息已成功送达 |
| ❌ 网络错误 | 无法连接 | 检查网络或 URL |
| ❌ HTTP 403/404 | 服务端拒绝 | 检查 URL 是否正确 |
| ❌ 超时 | 服务端无响应 | 检查服务端是否在线 |

## 5. 查看发送历史

### 5.1 进入历史页面
点击底部导航栏的 **📜 历史** 图标，切换到历史页面。

### 5.2 历史列表说明
每条记录以卡片形式显示：
- **左侧圆点**: 
  - 待办消息：红色 = P0高优，灰色 = P1一般，绿色 = P2低优
  - 反馈消息：青色圆点表示"意见反馈"
- **内容**: 摘要显示，超长的会自动省略。反馈消息显示为 `【意见反馈】反馈内容`
- **截止时间**: 仅待办消息显示（红色小字，可选）
- **时间**: 格式为 `年-月-日 时:分:秒`
- **状态标签**: 绿色"✓" 或 红色"✗"

### 5.3 历史记录数量
- 历史记录**最多保存10条**，包含待办消息和反馈意见
- 当新的记录加入时，最旧的历史记录会自动删除
- 如需保留更多记录，请自行备份

### 5.4 空状态
如果没有发送记录，页面中央会显示：📜 图标 + "暂无发送记录"

---

## 6. 提交意见反馈

### 6.1 反馈页面

1. 点击底部导航栏的 **✏️ 反馈** 图标，切换到意见反馈页面
2. 在输入框中输入您的意见或建议
3. 输入框下方会实时显示当前字数（最多100字）
4. 点击 **发送** 按钮提交

### 6.2 字数限制
- 反馈内容最多 **100 个字**
- 超出限制后输入框会变为红色警告状态
- 超过100字的内容输入将无效

### 6.3 反馈发送机制
- 反馈意见会自动附加 **"意见反馈"** 关键字
- 使用固定的 DingTalk Webhook 地址发送，无需配置自己的 Webhook
- 即使您尚未配置待办项的 Webhook，也可以正常使用反馈功能
- 反馈发送后会保存到本地历史记录中

### 6.4 查看反馈历史
- 在历史页面可以看到已发送的反馈意见
- 反馈记录以青色圆点标识，并显示 `【意见反馈】` 前缀

---

## 7. 修改 Webhook 地址

1. 点击任意页面上方的 **⚙ 设置** 按钮
2. 在对话框中修改 URL
3. 点击 **保存**

设置弹窗底部会显示当前版本号，例如：**版本：1.0.4**

## 8. 数据管理

### 8.1 数据存储位置
| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| Webhook URL | 本地偏好设置 (DataStore) | 随应用备份恢复 |
| 发送历史 | 本地 SQLite 数据库 (Room) | 不随云备份 |

### 8.2 数据安全
- 所有数据均存储在设备本地
- 不会上传到任何第三方服务器
- Webhook 地址仅保存在您自己的设备上

## 9. 常见问题（FAQ）

**Q: 为什么点击发送按钮没反应？**
> A: 请检查是否已配置 Webhook 地址。若"发送"页面顶部显示橙色提示卡片，请点击 ⚙ 设置并配置 Webhook 地址，确保待办项内容不为空。

**Q: 提示"发送成功"但对方没收到？**
> A: 请确认 Webhook URL 正确、服务端正常运行，并且服务端已正确解析 JSON 请求体。

**Q: 如何清除历史记录？**
> A: 当前版本历史记录自动管理，最多保留10条。超出后旧记录会自动删除。如需清除所有数据，请前往系统设置 → 应用 → WebhookSender → 清除存储。

**Q: 为什么历史记录变成了10条？**
> A: 为了控制存储占用，App 设计为最多保留最近10条发送记录（包括待办项和反馈意见），超出后自动清理最旧的记录。

**Q: 未配置 Webhook 能发反馈吗？**
> A: 可以。反馈页面使用固定的远程地址，无需配置 Webhook 即可使用。

**Q: 支持哪些 Webhook 服务？**
> A: 任何能接收标准 HTTP POST 请求、Content-Type 为 application/json 的服务均可。

**Q: 发送的 JSON 格式是怎样的？**
> A: 请求体格式如下（以待办项为例）：
```json
{
  "keyword": "todos",
  "content": "您输入的工作待办项",
  "priority": "P1一般",
  "deadline": "2026-06-15",
  "text": "todos 【P1一般】\n截止时间：2026-06-15\n您输入的工作待办项",
  "timestamp": 1717826400000
}
```
- 若未填写截止时间，deadline 为空字符串，text 中不含截止时间行
