# WebhookSender 使用说明书

**版本**: 1.0  
**适用平台**: Android 8.0+ (API 26+)

---

## 1. 应用简介

**WebhookSender** 是一款轻量级的 Android 消息发送工具。它可以将您输入的关键字和正文内容，以 JSON 格式发送到您指定的 Webhook 地址，适用于对接各类通知机器人、消息推送服务和自建 Webhook 服务。

---

## 2. 安装与启动

### 2.1 安装
1. 从开发环境构建 APK 包（使用 Android Studio 或 Gradle 命令）
2. 将 APK 文件传输到 Android 设备
3. 在设备上打开 APK 进行安装
4. （如系统提示）允许安装未知来源应用

### 2.2 启动
安装完成后，在应用列表中找到 **WebhookSender** 图标并点击启动。

---

## 3. 初次使用

### 3.1 配置 Webhook 地址
首次启动后，请先配置 Webhook 地址：

1. 点击右上角的 **⚙ 设置** 按钮
2. 在弹出的对话框中输入您的 Webhook URL，例如：
   - 企业微信机器人: `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY`
   - 钉钉机器人: `https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN`
   - 飞书机器人: `https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_KEY`
   - 自建服务: `https://your-domain.com/api/webhook`
3. 点击 **保存** 按钮
4. 返回主界面，状态指示器变为蓝色，显示 **"Webhook 已配置"**

> ⚠️ **注意**: 发送消息前必须先配置 Webhook 地址，否则无法发送。

---

## 4. 发送消息

### 4.1 操作流程

1. 在底部导航栏中切换到 **📤 发送** 页面
2. 在 **关键字** 框中输入消息关键字（如一行的标题，可选）
3. 在 **正文** 框中输入要发送的详细内容
4. 点击 **📤 发送** 按钮
5. 等待发送完成，查看结果提示

### 4.2 发送按钮状态
| 状态 | 含义 | 操作 |
|------|------|------|
| 蓝色可点击 | 可以发送 | 点击发送 |
| 灰色禁点 | Webhook 未配置 | 先点击 ⚙ 配置 |
| 带旋转圈 | 发送中 | 请等待 |

### 4.3 结果提示

| 提示 | 含义 | 建议 |
|------|------|------|
| ✅ 发送成功 | HTTP 200 | 消息已成功送达 |
| ❌ 网络错误 | 无法连接 | 检查网络或 URL |
| ❌ HTTP 403/404 | 服务端拒绝 | 检查 URL 是否正确 |
| ❌ 超时 | 服务端无响应 | 检查服务端是否在线 |

---

## 5. 查看发送历史

### 5.1 进入历史页面
点击底部导航栏的 **📜 历史** 图标，切换到历史页面。

### 5.2 历史列表说明
每条记录以卡片形式显示：
- **左侧色带**: 绿色 = 成功，红色 = 失败
- **关键字**: 加粗显示（若无则为"（无关键字）"）
- **正文**: 摘要显示，超长的会自动省略
- **时间**: 格式为 `年-月-日 时:分:秒`
- **状态标签**: 绿色"✓ 成功" 或 红色"✗ 错误原因"

### 5.3 空状态
如果没有发送记录，页面中央会显示：
- 📜 图标 + "暂无发送记录"

---

## 6. 修改 Webhook 地址

1. 点击任意页面上方的 **⚙ 设置** 按钮
2. 在对话框中修改 URL
3. 点击 **保存**

---

## 7. 数据管理

### 7.1 数据存储位置
| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| Webhook URL | 本地偏好设置 | 随应用备份恢复 |
| 发送历史 | 本地 SQLite 数据库 | 不随云备份 |

### 7.2 数据安全
- 所有数据均存储在设备本地
- 不会上传到任何第三方服务器
- Webhook 地址仅保存在您自己的设备上

---

## 8. 常见问题（FAQ）

**Q: 为什么点击发送按钮没反应？**
> A: 请检查是否已配置 Webhook 地址。若状态指示器为橙色，请点击 ⚙ 设置。

**Q: 提示"发送成功"但对方没收到？**
> A: 请确认 Webhook URL 正确、服务端正常运行，并且服务端已正确解析 JSON 请求体。

**Q: 如何清除历史记录？**
> A: 当前版本暂不支持手动删除，所有记录会自动保留直到应用数据被清除。

**Q: 支持哪些 Webhook 服务？**
> A: 任何能接收标准 HTTP POST 请求、Content-Type 为 application/json 的服务均可。

**Q: 发送的 JSON 格式是怎样的？**
> A: 请求体格式如下：
```json
{
  "keyword": "您输入的关键字",
  "content": "您输入的正文",
  "text": "关键字\n正文",
  "timestamp": 1717826400000
}
```
